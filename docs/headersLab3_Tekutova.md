## Лабораторная работа №3. Работа с Markdown-разметкой и базовое использование LaTeX в документации проекта

#### Markdown (Маркдаун) — это легкий язык разметки для форматирования текста с помощью простых символов (вроде * для курсива и ** для жирного шрифта), созданный для удобства чтения и написания, который легко преобразувать в HTML или другие форматы, широко используется на GitHub, Reddit, в документации и редакторах.
#### Основные сферы применения Markdown:
##### - техническая документация и IT
##### - веб-разработка и контент
##### - мессенджеры и форумы
##### - заметки и управление проектами
##### - создание книг и презентаций   

#### Markdown нужен разработчикам как стандарт де-факто для написания технической документации и README-файлов, потому что он сочетает в себе визуальную чистоту обычного текста и возможности полноценной верстки. В отличие от тяжелых форматов или чистого HTML, Markdown позволяет быстро форматировать текст, добавлять код, таблицы и ссылки без использования мыши и сложных редакторов, сохраняя при этом идеальную читаемость исходного файла даже без рендеринга. Для программистов критически важна совместимость этого формата с системами контроля версий вроде Git: поскольку файл остается текстовым, разработчики могут легко сравнивать изменения в документации (diff) так же, как и в программном коде. Кроме того, Markdown универсален и легко конвертируется в HTML, PDF или форматы для мобильных приложений, что делает его идеальным инструментом для ведения баз знаний, составления отчетов и общения на профессиональных платформах вроде GitHub и Stack Overflow.
